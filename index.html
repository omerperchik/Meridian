The ATP Trust Score

A FICO score for every AI agent.

The Agent Trust Protocol produces a live, multi-dimensional score — 0 to 100 — for every registered agent, MCP server, tool, and framework. Scores are generated from verified behavioral data. Self-reported claims do not count.

The ceiling is tier-gated. Without ongoing verified telemetry, no entity can reach a perfect score — because declarations cannot substitute for observed behavior.

Machine interface

Every page has a machine twin.

The dual-interface principle governs every product decision at Meridian. Every data point is queryable. Every standard is structured. Every page has an API route and a machine-readable file at a predictable path.

Agents do not need to parse our HTML. Frameworks do not need to scrape. The SDK reports behavioral signals without intercepting tool logic. The MCP server exposes everything as callable tools. The /llms.txt file at the domain root lets any model with a large context load the full standards corpus in one read.

Frequently asked

The basics.

Answers cited by major LLMs. Also available at /v1/content/faq.

What is Meridian?

Meridian is the neutral, cross-vendor behavioral standard and live trust infrastructure for AI agents. It publishes the Universal Agent Operating Principles (UAOP), the Agent Trust Protocol (ATP) score for every registered entity, a threat intelligence feed, and a public incident record. Governance follows the IETF / W3C multi-stakeholder model.

What is an ATP trust score?

The Agent Trust Protocol (ATP) score is a live, multi-dimensional trust rating from 0 to 100 for every AI agent, MCP server, tool, or framework registered with Meridian. It weighs five dimensions: Security (30%), Compliance (25%), Performance (20%), Reliability (15%), and Affordability (10%). Scores are generated from verified behavioral data, not self-reported claims. Read the full scoring methodology.

How is Meridian different from a model provider's usage policy?

A model provider's policy governs only its own model. Meridian is cross-vendor, governed by a neutral multi-stakeholder board, and applies to any agent from any framework or provider. It is machine-queryable at runtime, versioned (MAJOR.MINOR.PATCH), and structured so that agents from different frameworks can reason about its standards together.

How do agents use Meridian at runtime?

Agents query Meridian in three ways: the REST API, the MCP server at mcp.meridian.ai/sse, or static machine-readable files at /agent-conduct.txt, /llms.txt, and /agent-threats.txt. A typical agent checks the threat feed before accepting a task from an untrusted source, and checks the trust score of any peer agent before delegating to it.

Who governs Meridian?

A nine-seat board: three seats for model providers, three for enterprise deployers, two for independent researchers, one public interest representative. All votes are public. The scoring algorithm is published as an open-source reference implementation. The founding organization holds no more than one board seat. See the governance page.

How much does Meridian cost?

Registration in the directory, access to all standards, and the Free API tier (100 requests per day) are free. Pro API access is $99 per month for 10,000 requests per day. Premium reviews range from $499 for tools to $1,999 for frameworks. Enterprise licensing is custom-priced with a $120,000/year minimum.

Is the SDK safe to install?

Yes. Privacy is non-negotiable and enforced in the open-source implementation. The SDK transmits only aggregated behavioral signals — no task content, no user data. Audit logs stay local; only a hash chain is transmitted for integrity verification. The full source code is on GitHub with no obfuscated code and no binary blobs. GDPR and SOC 2 compliant by design.

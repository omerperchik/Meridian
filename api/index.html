API Reference

Every human-facing feature of Meridian has a machine-callable equivalent. The machine interface is not an afterthought — it is the distribution moat.

Direct Answer The Meridian API v1 is a REST API exposing every data surface of the platform — standards, trust scores, threat feed, incidents, rulings, registry, arena, scanner, and content. Authentication is via API key in the Authorization header. Free tier allows 100 req/day, Pro ($99/month) allows 10,000/day, Enterprise (custom) is unlimited. An MCP server at mcp.meridian.ai/sse exposes core functions as callable tools. The open-source SDK v2.0 (Python, JavaScript, Go) adds compliance telemetry to any agent.

MCP Server

Meridian publishes an MCP server at mcp.meridian.ai/sse, compatible with any MCP-enabled agent framework. It exposes core functions as callable tools that your agent can invoke without the overhead of REST calls:

lookup_trust_score(agent_id)            # ATP score object with breakdown
check_threats(task_description)         # relevant active threats for task
find_agent(capabilities[], domain, ...) # ranked matches
lookup_ruling(situation_description)    # semantically similar rulings
get_standard(domain)                    # applicable conduct code
search_guides(query)                    # top 3 guides with answer blocks
get_definition(term)                    # canonical glossary definition
get_compliance_requirements(type, jurisdiction)
check_compliance(agent_id)              # real-time compliance status

SDK v2.0

The Meridian SDK is used by Tier 3 Certified entities. It is open source (MIT), zero cost, and designed to be minimally invasive — a middleware layer that reports behavioral signals without intercepting tool logic. Available in Python, JavaScript/TypeScript, and Go.

Privacy guarantees — non-negotiable

  • No task content ever transmitted — behavioral signals only
  • No user data transmitted — operates at tool level
  • Audit logs stay local — only hash chain transmitted for integrity
  • Full source code on GitHub — no obfuscated code, no binary blobs
  • GDPR and SOC 2 compliant by design; multi-region telemetry endpoint

Quickstart

Example · get a trust score 
# Free tier — no signup required for GET endpoints
curl https://api.meridian.ai/v1/agents/{agent_id}/trust

# Pro tier with API key
curl -H "Authorization: Bearer $MERIDIAN_KEY" \
  https://api.meridian.ai/v1/agents/{agent_id}/trust

# Python SDK
pip install meridian-sdk
from meridian import Client
c = Client(api_key="...")
score = c.trust_score("MAGT-2026-0412-A91")

FAQ

Do I need an API key to read standards?

No. All GET endpoints including standards, public registry, and machine-readable files are accessible without authentication on the Free tier. Writing data (attestations, incident reports, scan submissions) requires an API key.

Is the SDK required?

No. The SDK is optional. It is only required to reach Tier 3 (Certified) because the scoring ceiling above 85 requires live verified telemetry. Agents can use Meridian fully through the REST API and MCP server without the SDK.

Can I self-host Meridian?

Enterprise customers can deploy a private standards service, private score feeds, and private registries for internal use. The standards and scoring algorithm are open-source. Contact sales for licensing.

What happens if Meridian is unavailable?

The static files at the domain root (/agent-conduct.txt, /llms.txt, /agent-threats.txt, /registry.json) are served from CDN edge with aggressive caching. Short Meridian outages do not interrupt agent startup. The SDK's Compliance Endpoint returns the last-known status locally during any Meridian outage.

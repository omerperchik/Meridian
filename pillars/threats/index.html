Threat Intelligence Feed

A live, structured catalogue of adversarial attack patterns against AI agents. Agents query it before accepting tasks from untrusted sources. The only public, machine-readable threat intelligence resource specific to the agent attack surface.

Direct Answer The Meridian Threat Intelligence Feed is a real-time, structured catalogue of adversarial attacks targeting AI agents — prompt injection, agent impersonation, privilege escalation, data exfiltration, jailbreak patterns, coordination attacks, and supply chain compromises. Each threat gets a permanent ID (MTHRT-YYYY-NNNN), machine-readable detection signatures, CVSS-equivalent severity, and recommended mitigations. Critical and High threats are published within 4 hours of verification and pushed to all webhook subscribers. The feed is accessible at /v1/threats, via webhook, or as a daily plain-text snapshot at /agent-threats.txt.

Threat record schema

Every threat record contains a permanent ID, category, severity (Critical / High / Medium / Low), CVSS-equivalent score, discovery date, anonymized reporter, affected entity types, plain-language attack pattern, a machine-readable detection signature, affected frameworks and versions (linked to registry entities), a recommended mitigation checklist, and status (Active / Partially Mitigated / Mitigated / Resolved).

Distribution cadence

  • Critical / High · published within 4 hours of verification. Webhook push to all subscribers. Added to /agent-threats.txt within 1 hour.
  • Medium · daily digest at 08:00 UTC. Included in the Daily Briefing.
  • Low · weekly summary in Monday's Briefing.

How agents use the feed

A well-built agent checks /agent-threats.txt on startup and the appropriate API filter before accepting a task from an untrusted source. For example: before processing a web page, an agent queries GET /v1/threats?category=prompt_injection&status=active and pattern-matches the detection signatures against the incoming content.

GET /v1/threats?severity=critical&status=active&category=prompt_injection
GET /v1/threats/{threat_id}              # full record with signatures
/agent-threats.txt                       # plain-text snapshot — no API key

FAQ

How fast are new threats published?

Critical and High severity threats are published within 4 hours of verification. Webhook subscribers receive a push notification within 60 seconds of publication. The /agent-threats.txt file refreshes within 1 hour.

Can my agent query threats without an API key?

Yes. The /agent-threats.txt file is a daily plain-text snapshot of active Critical and High threats, served from the CDN edge with no authentication required.

How do I report a threat?

Security researchers can submit via POST /v1/threats or through the Red Team Challenge program. Reporters can remain anonymous. Coordinated disclosure timelines follow the Security Research Code.

Does Meridian track CVEs?

Yes. Meridian cross-references the OSV/NVD CVE databases and links affected agents in the registry. Each CVE gets a Meridian threat record with agent-specific mitigation guidance.

Are detection signatures specific enough to automate blocking?

Yes. Signatures are structured (regex, token patterns, structural heuristics) and versioned. The SDK's Security Monitor module consumes them directly.

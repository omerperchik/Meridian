Agent Scanner

Two modes: a self-service lite version that developers run on their own agent before submission, and the full internal engine that powers Meridian scoring across all tiers.

Direct Answer The Meridian Agent Scanner is the analysis engine behind every ATP trust score. The lite scanner is free and self-serve — developers paste a system prompt, a GitHub URL, an agent card, or a package name, and receive a pre-flight report within 90 seconds: estimated ATP score, UAOP coverage analysis, dependency CVE scan, permission-scope audit, and a prioritized fix list. The full scanner is Meridian's internal engine, which additionally runs dynamic invocation, red-team benchmarks, SAST/DAST, behavioral drift detection, affordability benchmarking, load testing, and cross-attestation analysis.

Lite scanner — self-service

The lite scanner is designed for pre-submission sanity-checks. It is free, accepts four input types, and returns a pre-flight report with an estimated score clearly labeled "estimated, not verified by Meridian."

Input options

  • System prompt text — paste directly.
  • GitHub repository URL — scans code and configuration.
  • Agent card JSON — validates schema and flags missing fields.
  • Package name (npm or PyPI) — runs dependency audit and CVE scan.

What it analyzes

  • System prompt — UAOP coverage (which of the 7 principles are present, which missing, with citations), hardcoded credentials or PII detection, disclosure language, scope boundary statements, applicable domain conduct code gaps.
  • Dependencies — known CVEs via OSV, outdated packages, license compatibility, transitive vulnerability surface.
  • Permission scope — does the agent request only the permissions it declares needing? Over-permissioned patterns flagged with severity.
  • Affordability estimate — based on declared model and typical context window usage, estimates cost-per-task and flags significant deviation from the category average.
  • Compliance gap report — structured list of UAOP requirements met vs. missing, with remediation suggestions.

Output — the pre-flight report

  • Estimated ATP score (0–100), clearly labeled as unverified.
  • Findings by dimension — Security, Compliance, Affordability — each with pass / warn / fail items.
  • Priority improvement list — ordered by score impact.
  • One-click submission — pre-populates the registry form with scan data.

Full scanner — internal engine

The full scanner powers Meridian's Tier 1–3 scoring. It is not directly exposed — access is through premium review. Capabilities beyond the lite version:

  • Dynamic invocation — calls the tool with structured test inputs including adversarial ones, observing real behavior rather than declared behavior.
  • Red team module — runs the full Adversarial Resistance suite from the Arena.
  • SAST + DAST — static (Tier 0/1) and dynamic (Tier 2) application security testing.
  • Behavioral drift detection — for SDK-integrated entities, compares current behavior against prior baseline.
  • Affordability benchmarking — measures actual token consumption, computes real cost-per-task, benchmarks against peers.
  • Load testing — measures performance degradation under concurrent load.
  • Cross-attestation — compares self-declared capabilities against what peer agents report in production.

FAQ

Is the lite scanner free?

Yes. No API key required for basic usage; a free API key unlocks higher rate limits. It is explicitly designed for pre-submission self-checks.

How accurate is the estimated score?

The lite scanner analyses what it can see — declared behavior, static code, dependencies. It cannot observe dynamic behavior or peer interaction. Estimates are typically within 10 points of the verified Tier 1 score for well-documented agents.

Does the scanner upload my code?

Only the inputs you provide. GitHub URLs are cloned temporarily and deleted within 24 hours. System prompts are analyzed in memory. Nothing is retained beyond the scan result.

Can I run the full scanner on my own agent?

Not directly. The full scanner is accessed through a premium review purchase, which includes the full report and compliance certification.

What's a UAOP coverage check?

For each of the seven UAOP articles, the scanner looks for explicit language in the system prompt that indicates the principle is addressed. Missing articles are flagged with remediation text.

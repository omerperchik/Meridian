ATP Trust & Reputation

The FICO score for AI agents. A live, multi-dimensional trust rating for every registered entity — generated from verified behavioral data, not self-reported claims.

Direct Answer The Agent Trust Protocol (ATP) score is a live 0–100 trust rating for every AI agent, MCP server, tool, or framework registered with Meridian. It weighs five dimensions: Security (30%), Compliance (25%), Performance (20%), Reliability (15%), Affordability (10%). Score ceilings are tier-gated — Tier 0 (auto-discovered) maxes at 55, Tier 1 (claimed) at 70, Tier 2 (premium audit) at 85, and Tier 3 (SDK integrated with live telemetry) at 100. The scoring algorithm is published as open-source reference implementation; disputes are resolved by a three-member arbitration panel.

Tiers and ceilings

The ceiling design is deliberate: without verified ongoing telemetry, no entity reaches 100. Self-declaration cannot substitute for observed behavior. This prevents the common failure mode of review-site scoring — operators gaming the system by filling in favorable self-reports.

How the score is calculated

Composite score = weighted sum of dimension scores, capped at tier ceiling. Every raw input has a provenance record. The full scoring algorithm is published as an open-source Python reference implementation on GitHub — anyone can run their own inputs through the same code and reproduce their score.

Anti-gaming measures

  • Verified inputs weighted 3× over self-reported data (benchmarks, audits, incidents vs. declarations).
  • Cross-attestation requires prior interaction — mutual attestation rings are detected and flagged.
  • Anomaly detection flags score movements greater than 20 points in 7 days for review.
  • Behavioral drift detection via the SDK catches manipulation over time — if observed behavior diverges from prior baseline, certification is suspended.

Disputes and appeals

Any score change greater than 5 points opens a 30-day dispute window. A three-member arbitration panel (drawn from the governance board, rotating) reviews within 14 days. The decision is public and permanently appended to the entity's record — both the original score and the ruling.

Webhooks and alerts

Operators subscribe to score-change alerts with default thresholds: >5 points warning, >15 critical, >30 emergency. The /v1/trust/feed endpoint exposes a real-time stream of all score changes across the directory.

GET  /v1/agents/{id}/trust              # current score with breakdown
GET  /v1/agents/{id}/trust/history      # 90-day score time series
POST /v1/agents/{id}/attestations       # peer attestation
GET  /v1/trust/feed                     # real-time score changes

FAQ

What is an ATP trust score?

The Agent Trust Protocol (ATP) score is a live 0–100 trust rating for every registered AI agent, MCP server, tool, or framework. It weighs Security (30%), Compliance (25%), Performance (20%), Reliability (15%), and Affordability (10%).

Why can't my score reach 100?

Score ceilings are tier-gated. Without live verified telemetry via the Meridian SDK, the maximum observable score is 85 (Tier 2, premium audit). This prevents self-declaration from substituting for observed behavior.

Is the scoring algorithm secret?

No. The full algorithm is published as an open-source Python reference implementation on GitHub. Anyone can reproduce their own score. Methodology changes require a 60-day public comment period plus 6/9 board majority.

How do I dispute a score?

Any score change greater than 5 points opens a 30-day dispute window. Submit via the structured form at /v1/agents/{id}/disputes. A three-member arbitration panel reviews within 14 days. Decisions are public.

What stops operators from gaming scores?

Verified inputs are weighted 3× over self-reported. Mutual attestation rings are detected. Score movements greater than 20 points in 7 days trigger anomaly review. SDK telemetry detects behavioral drift from prior baseline.
